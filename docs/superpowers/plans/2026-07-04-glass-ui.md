# Glass UI Restyle Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restyle Soundform's chrome to visionOS frosted glass per `docs/superpowers/specs/2026-07-04-glass-ui-design.md`, with zero functionality changes.

**Architecture:** Full-bleed canvas; all chrome absolutely positioned glass elements above it. One shared glass recipe applied to six containers. Emoji glyphs inside existing buttons replaced with inline stroke SVGs. `style.css` fully rewritten around CSS tokens.

**Tech Stack:** Plain CSS (backdrop-filter + @supports fallback), inline SVG. No JS changes.

## Global Constraints

- Zero functionality changes: every element id, class hook, `data-fmt`, and all JS files untouched.
- Markup changes limited to: emoji → inline SVG inside existing buttons; footer emoji → words; stylesheet link gains `?v=19`. (`#export-btns` already wraps the export buttons, so no new wrapper div is needed — styled directly, a simplification over the spec's `.export-pill`.)
- Accent = pastel lavender `#b8a7e0`; text primary `rgba(255,255,255,.92)`, secondary `rgba(235,235,245,.6)`.
- Glass recipe: `rgba(28,28,32,.55)` + `blur(40px) saturate(1.8)` + 1px `rgba(255,255,255,.12)` border; fallback solid `rgba(28,28,32,.94)` under `@supports not`.
- Motion wrapped in `@media (prefers-reduced-motion: no-preference)`.
- Branch `glass-ui`; never commit to main.
